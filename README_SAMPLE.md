## 项目概览

本项目是一套完整的 **AI 数字人口播视频自动化生成流水线**，将文案提取、AI 仿写、语音克隆合成、数字人驱动、视频后期制作及多平台一键发布整合为统一工作流。用户只需输入一个视频链接，系统即可自动完成从文案到成片发布的全流程操作。

### 核心能力

| 环节 | 能力 |
|------|------|
| 📝 文案处理 | 视频链接提取文案 → AI 智能仿写/指令仿写 |
| 🎙️ 语音合成 | CosyVoice 语音克隆与 TTS |
| 🧑 数字人生成 | TuiliONNX 数字人驱动（双版本） |
| 🎬 视频后期 | 字幕生成、背景音乐、AI 封面图 |
| 📤 多平台发布 | 抖音 / 小红书 / 视频号一键发布 |

## 各环节模型详解

### 1. 文案提取 — faster-whisper（本地）

| 属性 | 说明 |
|------|------|
| **模型名称** | faster-whisper-small（CTranslate2 加速） |
| **用途** | 从短视频链接中提取音频，将语音转为文字 |
| **运行方式** | 🏠 **纯本地运行** |
| **输入** | 视频 URL（抖音/小红书/视频号） |
| **输出** | 原始口播文案文本 |
| **显存占用** | ~1.5 GB（int8 量化） |
| **所属模块** | `local_models/asr_engine.py` |
| **技术特点** | 比原版 Whisper 快 4 倍，显存省 38%，支持 99 种语言 |

### 2. 文案仿写 — Qwen2.5（本地）

| 属性 | 说明 |
|------|------|
| **模型名称** | Qwen2.5-3B-Instruct（阿里通义千问） |
| **用途** | 对原始文案进行智能改写，生成原创口播内容 |
| **运行方式** | 🏠 **纯本地运行**（Ollama / llama.cpp） |
| **输入** | 原始文案 + 用户指令（可选） |
| **输出** | 仿写后的口播文案 |
| **显存占用** | ~3.5 GB（int4 量化）；低配自动降级为 1.5B(~1.8GB) 或 0.5B(~0.8GB) |
| **两种模式** | ① AI 自动仿写：保留核心意思，优化语言表达；② 根据指令仿写：用户自定义 prompt 精准控制风格和内容 |
| **所属模块** | `local_models/llm_engine.py` |
| **降级策略** | 8GB+ → 3B / 6GB+ → 1.5B / <6GB → 0.5B / 无GPU → CPU 模式 |

### 3. 语音合成 — CosyVoice-300M（本地）

| 属性 | 说明 |
|------|------|
| **模型名称** | CosyVoice-300M-SFT（阿里通义实验室） |
| **用途** | 将仿写文案合成为自然语音，支持声音克隆 |
| **运行方式** | 🏠 **纯本地运行**（API: `localhost:9880`） |
| **输入** | 文本 + 参考音频（声音克隆） |
| **输出** | WAV/MP3 音频文件 |
| **显存占用** | ~2.0 GB（模型仅 300MB） |
| **服务启动** | `start_cosyvoice_api.py` 或 `start_cosyvoice_service.py` |
| **所属模块** | `local_models/tts_engine.py` |
| **技术特点** | 3 秒零样本声音克隆，支持中/英/日/粤/韩，情感控制 |

### 4. 数字人视频生成 — MuseTalk / Wav2Lip（本地）

| 属性 | 说明 |
|------|------|
| **模型名称** | MuseTalk（腾讯音乐 Lyra Lab）/ Wav2Lip（降级备选） |
| **用途** | 根据音频驱动数字人形象，生成口播视频 |
| **运行方式** | 🏠 **纯本地运行** |
| **输入** | 音频文件 + 人脸视频/图片 |
| **输出** | 口播视频 |
| **显存占用** | MuseTalk ~4.0 GB / Wav2Lip ~2.5 GB / Wav2Lip Lite ~1.5 GB |
| **所属模块** | `local_models/lipsync.py` |
| **技术特点** | 自动分级：8GB+ → MuseTalk（30fps+ 实时高画质）；6GB+ → Wav2Lip；<6GB → Wav2Lip Lite |
| **降级策略** | MuseTalk 加载失败时自动降级为 Wav2Lip |

### 5. 视频后期 — FFmpeg + ImageMagick

| 属性 | 说明 |
|------|------|
| **字幕生成** | 两种模式：普通字幕（基础文字叠加）和高级字幕（逐词高亮、动态效果） |
| **背景音乐** | 支持随机选取内置 BGM / 手动选择 / 用户自定义上传 |
| **封面图生成** | AI 自动生成模式 + 手动模式，支持自定义字体、颜色、关键词高亮 |
| **核心工具** | FFmpeg（音视频处理） + ImageMagick（图像处理） |
| **所属模块** | `video_tools/subtitle_utils.py`, `utils/video_cover_image.py` |

### 6. 多平台发布 — Playwright 自动化

| 属性 | 说明 |
|------|------|
| **技术方案** | Playwright + Chrome DevTools Protocol |
| **支持平台** | 抖音、小红书、视频号（蝴蝶号） |
| **浏览器要求** | Chrome 需开启远程调试端口 `--remote-debugging-port=9222` |
| **所属模块** | `video_tools/publisher.py` |
| **技术特点** | 基于 Playwright 的浏览器自动化，模拟人工操作进行内容发布 |

## 环境与部署

### 系统要求

- **操作系统**：Windows 10/11（当前仅支持 Windows）
- **Python**：3.12（Conda 环境 `avatar`）
- **Chrome**：需要安装 Chrome 浏览器
- **FFmpeg**：需要安装并配置环境变量
- **ImageMagick**：7.1.1 版本
- **GPU**：推荐 NVIDIA 显卡 6GB+ 显存（支持 CUDA）

### 显存分级与模型选择

系统会根据 GPU 显存自动选择最优模型方案，无需手动配置：

| GPU 显存 | ASR | LLM 仿写 | TTS | 口型合成 |
|----------|-----|----------|-----|---------|
| **8GB+**（RTX 5060） | Whisper small | Qwen2.5-3B (int4) | CosyVoice-300M | **MuseTalk** ⭐ |
| **6GB+**（RTX 3060） | Whisper small | Qwen2.5-1.5B (int4) | CosyVoice-300M | Wav2Lip |
| **4GB+**（GTX 1050Ti） | Whisper tiny | Qwen2.5-0.5B (int4) | CosyVoice-300M | Wav2Lip Lite |
| **CPU Only** | Whisper tiny | Qwen2.5-1.5B (GGUF) | CosyVoice-300M | ❌ 不可用 |

> **核心设计**：所有模型 **分时加载**，用完后立即释放显存。在 8GB 显卡上，实际峰值显存 ≈ 4.0 GB（MuseTalk）。

### 前置依赖

| 依赖 | 获取方式 |
|------|----------|
| 完整项目源码 | 百度网盘（见 `代码地址.txt`） |
| 模型文件 & 运行环境 | 百度网盘（见 `使用前必装.txt`） |
